# Code of Conduct

QATRA Engine is an open-source project. Everyone is welcome to use, test, review, criticize, and contribute to the project.

## Expected Behavior

- Be respectful and constructive.
- Provide clear technical feedback when opening issues or pull requests.
- Assume good intent.
- Focus criticism on code, design, documentation, and testing behavior.
- Help make the project useful for the QA and automation community.

## Unacceptable Behavior

- Harassment, personal attacks, or abusive language.
- Discrimination or exclusionary behavior.
- Sharing private or sensitive information without permission.
- Using the project to attack, overload, or abuse third-party systems.

## Testing Ethics

When using QATRA against public or government websites, keep tests lightweight and ethical:

- Do not perform load testing without permission.
- Do not perform security testing without permission.
- Do not submit real forms or sensitive data without authorization.
- Prefer smoke tests, read-only checks, and local demo pages.

## Reporting Issues

Please open a GitHub issue with:

- QATRA version or commit hash
- Java version
- Browser and browser version
- Operating system
- Minimal reproduction example
- Logs or reports when possible

Thank you for helping improve QATRA Engine.
